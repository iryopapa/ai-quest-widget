# MANUSへの依頼: ゲーミフィケーションウィジェット連携

## 依頼概要

現在のAI教育カリキュラムサイトに、外部のゲーミフィケーションウィジェット（JavaScript）を連携させたいです。サイトの既存機能は一切変更せず、以下の2点のみ対応をお願いします。

---

## 依頼1: ウィジェットの読み込み（scriptタグ1行の追加）

サイトの全ページの `</body>` タグの直前に、以下のscriptタグを2つ追加してください。

```html
<script src="https://iryopapa.github.io/ai-quest-widget/ai-quest-widget.js"></script>
<script>
  AIQuest.init({
    chapters: [
      { id: 'ch1', title: '第1章のタイトル', items: 3 },
      { id: 'ch2', title: '第2章のタイトル', items: 4 },
      // ※ 章のタイトルと項目数はカリキュラムの内容に合わせて設定してください
    ]
  });
</script>
```

この追加により、画面右下にキャラクター育成ゲームのウィジェットが表示されます。
既存のサイトのデザインや機能には影響しません（Shadow DOMで隔離されています）。

---

## 依頼2: 学習イベントとウィジェットの連携

受講者が以下の学習アクションを完了したとき、ウィジェットに通知するJavaScriptを追加してください。

### 連携したいイベント一覧

| 学習アクション | 呼び出すコード | 説明 |
|---|---|---|
| 動画の視聴が完了したとき | `AIQuest.onVideoComplete()` | 動画プレーヤーの再生完了イベントに紐づけ |
| テキスト教材を最後まで読んだとき | `AIQuest.onTextComplete()` | ページ最下部へのスクロール完了、または「読了」ボタンのクリックに紐づけ |
| 小テストに合格したとき | `AIQuest.onQuizPass(false)` | 小テストの合格判定後に呼び出し |
| 小テストに満点で合格したとき | `AIQuest.onQuizPass(true)` | 全問正解で合格した場合に呼び出し |
| 章をクリアしたとき（次の章へ進むとき） | `AIQuest.onChapterComplete('ch1')` | 引数の 'ch1' は該当する章のIDに変更。依頼1のchapters配列のidと一致させてください |

### 連携の仕組み（技術的な説明）

各イベントの連携は、既存のイベント発生箇所に1行のJavaScript呼び出しを追加するだけです。

**例1: 動画プレーヤーの場合**

もし動画がHTMLの `<video>` タグで実装されている場合:
```javascript
document.querySelector('video').addEventListener('ended', function() {
  AIQuest.onVideoComplete();
});
```

もしYouTube埋め込みやVimeo等の外部プレーヤーを使用している場合は、そのプレーヤーの再生完了コールバックに `AIQuest.onVideoComplete()` を追加してください。

**例2: 小テスト合格の場合**

小テストの合格判定を行っているJavaScriptの中で、合格と判定された直後に追加:
```javascript
// 既存のコード（例）
if (score >= passingScore) {
  showPassMessage();  // 既存の合格処理
  
  // ↓ この1行を追加
  if (typeof AIQuest !== 'undefined') {
    AIQuest.onQuizPass(score === totalScore);  // 満点かどうかを判定
  }
}
```

**例3: 章クリア（次の章へ進む）の場合**

「次の章へ」ボタンのクリックイベント、または章の完了処理の中に追加:
```javascript
// 既存のコード（例）
function goToNextChapter(chapterId) {
  // 既存の遷移処理...
  
  // ↓ この1行を追加
  if (typeof AIQuest !== 'undefined') {
    AIQuest.onChapterComplete(chapterId);
  }
}
```

### 重要な注意点

- `if (typeof AIQuest !== 'undefined')` で囲むことで、ウィジェットが読み込めなかった場合でもエラーが発生しません
- 既存の処理の「後」に追加してください。既存の処理を変更・置換しないでください
- これらのコードはすべてフロントエンド（ブラウザ側）で動作します。サーバー側の変更は不要です

## 依頼3（任意）: 既存の経験値・ポイントとの統合

現在サイトに経験値やポイントの仕組みがある場合、それらをウィジェットのシステムに統合（置き換え）することも可能です。

**選択肢A: 両方残す（簡単）**
既存の経験値・ポイントはそのまま残し、ウィジェット側でも別途XP・コインを管理する。受講者は2つのポイントを持つことになりますが、実装は最も簡単です。

**選択肢B: ウィジェットに統合する（おすすめ）**
既存の経験値・ポイント表示を非表示にし、ウィジェットのXP・コインに一本化する。受講者にとってシンプルでわかりやすい体験になります。
この場合、既存の経験値表示のHTML要素に `style="display:none"` を追加するだけです。

## 確認したいこと

1. 動画プレーヤーはどのような技術で実装されていますか？（HTML5 video / YouTube埋め込み / Vimeo / その他）
2. 小テストはどのような仕組みで動いていますか？（WordPressプラグイン名 / カスタムJavaScript / その他）
3. 「次の章へ」の遷移はどのように実装されていますか？（ボタンクリック / 自動遷移 / その他）
4. 既存の経験値・ポイントはどのように管理されていますか？（JavaScript変数 / LocalStorage / サーバー側 / WordPressプラグイン）

上記の情報があれば、より具体的な連携コードを提示できます。

---

## 動作確認方法

連携設定後、以下で動作確認できます:

1. サイトにアクセスすると右下にキャラクター（たまご）が表示される
2. キャラクターをタップするとゲームパネルが開く
3. 動画を視聴完了すると「+20 XP, +5 コイン」の通知が出る
4. 小テストに合格すると「+30 XP, +15 コイン」の通知が出る
5. キャラクターが経験値に応じてレベルアップ・進化する
